# KazUTB Digital Library Platform - Canonical Project PRD

## Project Vision
A single, coherent, and clean design system project for the KazUTB Digital Library. This is the official "source of truth" project, encompassing public, member, librarian, and admin roles within a unified visual family.

## Design Principles (The "Locked" Style)
*   **Tone:** Calm, academic, editorial, institutional, and high-trust.
*   **Typography:** Serif-forward large headlines; clean sans-serif body/system text.
*   **Color Palette:** Deep navy / midnight ink (brand), restrained teal / archive (accent), soft light surfaces.
*   **Visual Language:** Minimal and premium card language, quiet but strong CTA rhythm, elegant spacing.
*   **Global Shell:** All pages share the same navbar, footer, and framing. No alternate shells.

## User Roles
1.  **Guest:** Public visitor (Home, Catalog, Book Details, Resources, About).
2.  **University User:** Students and Faculty (Member Dashboard, Shortlist, Digital Access).
3.  **Librarian:** Internal operator (Records, Assets, Circulation, Metadata).
4.  **Admin:** Oversight and system visibility.

The following screens define the canonical language:
1. Academic Discovery Hub
2. Research Workspace / Shortlist

4. About the Institution
5. Member Dashboard

7. Librarian Record Editor
8. Librarian Operations Center
9. Digital Asset & File Management
10. Book Cover & Media Management
11. Reader Access Clarification Hub
12. Circulation & Waitlist Manager
13. Metadata Quality & Curation Hub

## Missing Screens to Generate
1.  **Homepage:** Orientation and route-forward landing. No marketing excess.
2.  **Catalog:** Academic search and browse. Focused on filtering and availability.
3.  **Book Details:** Bibliographic information and access/shortlist actions for a single record.
4.  **Resources:** Directory for external licensed and research tools.
5.  **Admin Overview:** System-level oversight, sibling to librarian pages.

## Strict Constraints
*   **No Duplicates:** One canonical screen per page type.
*   **No Redesign:** Approved screens are locked.
*   **No Shell Innovation:** Use the existing navbar/footer/button family.
*   **Body-First:** Focus on page-specific content within the fixed shell.
*   **No Decorative Filler:** Every section must have a product purpose.
*   **Grounded Copy:** Short, realistic, library-appropriate wording. No "world-class" or "AI-powered" fluff.
