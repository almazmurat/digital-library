# design-system/pages/README.md

## Page-Specific Design System Overrides

- If a page (e.g., homepage, catalog, book detail) needs special design rules, create a `[page].md` file here.
- Each file should only include overrides or additions to the global rules in `../MASTER.md`.
- If no file exists for a page, use the global rules only.

**Example:**

- `homepage.md` — for homepage-specific layout or hero section rules
- `catalog.md` — for catalog grid or filter UI specifics

---

**How to use:**

- Always check for a `[page].md` file before starting work on a specific page.
- If you add a new override, document the reason and scope clearly.
