# Phase 2 Alerting and Failure Handling

| Scenario/Event                                           | Alert Type                     | Recipient / Owner                | Action                                                      | Escalation                                           | Notes                                 |
| -------------------------------------------------------- | ------------------------------ | -------------------------------- | ----------------------------------------------------------- | ---------------------------------------------------- | ------------------------------------- |
| CI job fails (backend-quality/browser-smoke/phase2 gate) | GitHub workflow failed status  | Repository maintainers, QA owner | Review failed job logs and uploaded artifacts               | If repeated across 2 runs, escalate to module owner  | Primary alerting mechanism in project |
| API smoke gate below threshold                           | Phase 2 quality gate fail      | QA lead, API owner               | Inspect phase2-api-tests.log and failing test IDs           | Open defect and block release branch merge           | Current status: fail                  |
| UI smoke detects 5xx on public page                      | Phase 2 quality gate fail      | QA lead, web owner               | Triage failing route and attach screenshot/context evidence | Escalate as production risk if public route affected | Current status: /news returned 500    |
| PHPUnit targeted suite below threshold                   | Phase 2 quality gate fail      | QA lead, backend owner           | Review failing tests and environment assumptions            | Escalate if fail persists in CI main branch          | Current status: fail                  |
| Coverage metric blocked                                  | Phase 2 quality gate warn      | QA lead                          | Mark metric as blocked with reason, avoid fabricated values | Escalate if still blocked by Phase 3 gate date       | Current status: warn                  |
| Flaky or timeout trend                                   | Phase 2 quality gate warn/fail | QA lead                          | Compare execution-time trend and rerun count                | Escalate when runtime breaches threshold twice       | Currently runtime within threshold    |
